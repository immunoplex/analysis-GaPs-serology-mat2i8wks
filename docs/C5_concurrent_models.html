SBA: Concurrent Within-Visit Antibody Analysis

This is a focused, expanded extract of the full config-driven serology pipeline. It runs only the concurrent (within-visit) association layer for the infant’s serum bactericidal activity (WHOLE_SBA), restricted to the 8-week infant timepoint (InfMon2), and it now covers the expanded feature set — FIM plus the FcγR binding and antigen-specific effector functions — on the untransformed (raw) log assay values. The total-IgG-standardised mode is deferred. The maternal-chain, clinical, and other pipeline parts are excluded.

The endpoint is set once (params$endpoint = "SBA"); switching to WT_IgG or PTNA is a one-line change to that parameter (or use the render_concurrent_all() helper in the appendix to knit all three at once).

Visit naming. All visits are recoded for clarity throughout: P02→PregEarly, P09→MatBirth, M00→CordBlood, M02→InfMon2, M04→InfMon4, M05→InfMon5, M09→InfMon9. This analysis is restricted to the 8-week infant timepoint (InfMon2).

Stratification. Primary axis is the maternal vaccination arm (TdaP vs TT); within each, results are reported pooled across infant arms and separately for each infant priming level (aP, wP).

Downstream use. The knitted HTML is intended for the summary engine; the Word document is for the analysis recipient. Inline “🧭 Maps to interpretation” callouts tie each block to the matching section of the written concurrent-analysis summary for this endpoint.

Concurrent Within-Visit Architecture (Expanded)

This part models the infant’s SBA against antibody features measured at the same infant visit, restricted to the 8-week timepoint (InfMon2). Relative to the original version it adds the FIM antigen, the FcγR binding features (FcgR2a, FcgR3b), and the antigen-specific effector functions (ADCD for all six antigens; ADCP and ADNP for PT/PRN/DT/TT only), on the untransformed (raw) log assay values. The total-IgG-standardised mode is deferred and not run here.

Stratification is maternal-arm primary (TdaP / TT), with infant splits (aP, wP) nested. Networks / NCT need ≥ 30 complete cases; underpowered cells are skipped and recorded in the participant accounting. Execution is staged (cheap → expensive) with live console logging.

conc_by_mode <- list()
for (md in DATA_MODES) {
  cat("\n## Mode: `", md, "`\n\n", sep = "")
  sx <- strata_by_mode[[md]]
  conc <- lapply(names(sx), function(k) {
    r <- run_blocks_x(sx[[k]]$data, stratum = sx[[k]]$label, mode = md)
    .info("mode=%-7s stratum=%-26s blocks fitted", md, sx[[k]]$label)
    r
  })
  names(conc) <- names(sx)
  conc_by_mode[[md]] <- conc
  for (m in cfg$maternal_arms) {
    cat("### ", m, " mothers\n\n", sep = "")
    show_stratum_blocks(conc[[m]], full = TRUE)
    for (i in cfg$infant_arms) {
      key <- paste0(m, "_", i)
      if (!is.null(conc[[key]])) {
        cat("#### ", sx[[key]]$label, "\n\n", sep = "")
        show_stratum_blocks(conc[[key]], full = FALSE)
      }
    }
  }

run_one_network <- function(dat, nodes, communities, groups_idx, group_colors,
                            title, tracker_label, mode, which) {
  d <- dat %>% dplyr::select(dplyr::all_of(nodes)) %>% tidyr::drop_na()
  nt_log(nt, "Concurrent", paste0(which, " network @ ", CONC_VISIT),
         title, tracker_label, nodes, dat)
  cat(sprintf("\n**%s** (n = %d)\n\n", title, nrow(d)))
  .info("mode=%-7s %s network: %s n=%d", mode, which, title, nrow(d))
  if (nrow(d) < NETWORK_MIN) { cat(sprintf("*n < %d; skipped.*\n\n", NETWORK_MIN)); return(NULL) }
  net <- tryCatch(bootnet::estimateNetwork(d, default = "EBICglasso", threshold = TRUE),
                  error = function(e) NULL)
  if (is.null(net)) { cat("*Network estimation failed.*\n\n"); return(NULL) }
  qgraph::qgraph(net$graph, groups = groups_idx, color = group_colors,
                 layout = "spring", labels = nodes, legend = TRUE,
                 legend.cex = 0.33, vsize = 7, title = title)
  br <- networktools::bridge(net$graph, communities = communities)
  bdf <- data.frame(Variable = names(br$`Bridge Strength`),
                    Bridge_Strength = round(br$`Bridge Strength`, 3),
                    Bridge_EI = round(br$`Bridge Expected Influence (1-step)`, 3),
                    row.names = NULL) %>% dplyr::arrange(dplyr::desc(Bridge_Strength))
  print(knitr::kable(bdf, caption = paste0("Bridge centrality — ", title))); cat("\n")
  bdf
}

      output_file   = sprintf("concurrent_%s.%s", ep, ext),

      envir         = new.env())   # fresh environment per endpoint
